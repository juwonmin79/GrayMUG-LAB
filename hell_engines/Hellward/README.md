# Hellward

Hellward is the experimental Ward attachment space for GrayMUG-LAB.

Purpose:

- Test LAB risk hint and calibration hint consumption without modifying Production Ward.
- Evaluate how Ward can read LAB context while preserving independent defense judgment.

Initial attachment candidates:

- LAB risk hint reference.
- Calibration confidence context.
- Alert payload risk metadata.

Rules:

- Do not copy Production Ward code into this folder.
- Do not replace Ward regime or emergency exit decisions.
- Do not alter BTC MA200, FNG, BEAR, RECOVERY, or emergency exit rules.
- Do not add automatic liquidation or service control logic.
